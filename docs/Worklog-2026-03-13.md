# Worklog

> Date: 2026-03-13

이 문서는 오늘 기준 학원 seed 수집 파이프라인 정비, 실제 적재, 공개 노출 보정까지의 결과를 기록한다.

## 1. 환경 정리

- Node 실행 환경을 `Node 20` 기준으로 맞췄다.
- Windows + WSL을 오가도 native dependency가 최대한 덜 깨지도록 `pnpm-workspace.yaml`에 `supportedArchitectures`를 추가했다.
- Prisma Client가 두 환경에서 모두 생성되도록 `apps/api/prisma/schema.prisma`에 Windows + Debian binary target을 반영했다.
- 로컬 실행 기준 DB는 `localhost:5433`, Redis는 `localhost:6379`를 사용한다.

## 2. 수집 파이프라인 변경

- [apps/api/src/scripts/academy-seed-sync.ts](/mnt/c/Users/RAN/edupick/apps/api/src/scripts/academy-seed-sync.ts)에 seed 수집 스크립트를 계속 확장했다.
- 수집 순서는 `NEIS -> data.go.kr 파일/URL 입력 -> 좌표 보강 -> Prisma upsert` 구조다.
- 기본 중복 판단 키는 `전화번호 우선`, 없으면 `정규화된 학원명 + 주소`다.
- 시스템 import 전용 강사 계정 `seed-import@edupick.local` 아래로 seed 학원을 적재한다.
- 이미 일반 강사 소유 학원이 같은 전화번호 또는 이름+주소로 존재하면 덮어쓰지 않고 중복으로만 집계한다.

### 2.1 좌표 보강

- Kakao Local은 앱에서 `OPEN_MAP_AND_LOCAL` 서비스가 비활성화되어 실제 사용이 불가능했다.
- 그래서 `KAKAO_REST_API_KEY`가 동작하지 않을 때 `NAVER_MAP_CLIENT_ID` + `NAVER_MAP_CLIENT_SECRET`으로 fallback 하도록 바꿨다.
- 현재 스크립트는 Kakao -> Naver 순서로 좌표 보강을 시도한다.

### 2.2 data.go.kr 입력 경로

- 원래는 JSON 입력만 처리했지만, 서울시 학원 데이터 파일을 바로 쓸 수 있게 확장했다.
- `--data-go-file`로 로컬 파일을 읽을 수 있다.
- JSON은 `DATA` 배열 구조도 인식하도록 보강했다.
- CSV는 CP949/EUC-KR 계열 인코딩까지 읽도록 처리했다.
- 서울시 오픈데이터의 한글/소문자 키(`pei_nm`, `road_nm_addr`, `reg_stts_nm`, `학원명`, `도로명주소` 등)도 현재 정규화 로직이 인식한다.

## 3. 실행 방법

루트 기준 dry-run:

```bash
pnpm --filter @edupick/api exec ts-node src/scripts/academy-seed-sync.ts \
  --dry-run \
  --max-records=50 \
  --data-go-file=data/서울시\ 학원\ 교습소정보.json \
  --output=tmp/academy-seed-preview.json
```

실제 적재:

```bash
pnpm --filter @edupick/api exec ts-node src/scripts/academy-seed-sync.ts \
  --max-records=2000 \
  --data-go-file=data/서울시\ 학원\ 교습소정보.json \
  --output=tmp/academy-seed-imported-2000.json
```

주요 옵션:

- `--regions=B10:강남구,B10:서초구`
- `--data-go-url=https://...`
- `--data-go-file=data/서울시\ 학원\ 교습소정보.json`
- `--data-go-file=data/서울시\ 학원\ 교습소정보.csv`
- `--skip-geocoding`
- `--skip-kakao-geocoding`
- `--output=tmp/academy-seeds.json`

환경변수:

- `NEIS_API_KEY` 필수
- `DATA_GO_KR_SERVICE_KEY`는 data.go.kr API 호출 시 필요
- `DATA_GO_KR_ACADEMY_API_URL` 또는 `--data-go-url` 중 하나가 있으면 data.go.kr API branch를 직접 호출할 수 있다
- `KAKAO_REST_API_KEY` 또는 `NAVER_MAP_CLIENT_ID` + `NAVER_MAP_CLIENT_SECRET`이 있으면 주소 기준 좌표 보강을 시도한다

## 8. 검증 메모

- `academy-seed-sync.ts`는 `ts-node --transpile-only` 기준으로 로드 확인했다.
- 실제 외부 API 호출은 `NEIS` + `NAVER geocoding` 기준으로 dry-run 및 upsert까지 수행했다.
- 전체 API `tsc`는 이 환경에서 timeout 없이 끝나는지까지는 확인하지 못했다.

## 9. 후속 보강

- `--skip-kakao-geocoding` 옵션이 전체 geocoding을 꺼 버리던 파싱 버그를 수정했다.
- 이제 `--skip-kakao-geocoding`을 사용하면 Kakao만 건너뛰고, `NAVER_MAP_CLIENT_ID` + `NAVER_MAP_CLIENT_SECRET`이 있으면 Naver geocoding fallback은 계속 동작한다.
- 주소 좌표 보강은 단일 주소 1회 조회 대신 `원본 주소 -> 괄호 제거 -> 쉼표 제거 -> 상세 층/호 제거 -> 행정구역 축약` 후보를 순차 재시도하도록 확장했다.
- Kakao keyword fallback도 첫 결과를 바로 쓰지 않고, `학원명 / 구 단위 지역 / 카테고리` 조합으로 재조회한 뒤 이름과 주소가 더 잘 맞는 문서를 우선 선택하도록 보강했다.
- 회귀 방지를 위해 [academy-seed-sync.spec.ts](/mnt/c/Users/RAN/edupick/apps/api/test/academy-seed-sync.spec.ts)와 [jest.config.js](/mnt/c/Users/RAN/edupick/apps/api/jest.config.js)를 추가했다.
- 이 환경에서는 전체 `jest`, `tsc --noEmit`가 timeout 안에 끝나지 않아, 핵심 내부 함수에 대한 `ts-node/register/transpile-only` 기반 sanity check로 옵션 파싱과 후보 생성/선택 로직을 확인했다.

## 12. 미해결 5건 판단

- 위 5건은 원문 주소, 상호명, `상호명 + 구` 기준으로 Kakao / Naver 조회를 각각 재확인했지만 모두 응답 `0건`이었다.
- 따라서 현재 단계에서의 미해결 원인은 geocoding 재시도 부족보다는 원본 데이터의 주소 불완전, 오기, 지도 플랫폼 미등록 가능성이 더 크다고 판단했다.
- 예시:
  - `압루정로 42길`은 도로명 오타 가능성이 있다.
  - `헌릉로`는 건물번호가 없어 주소 자체가 너무 넓다.
- 현재는 이 5건을 억지로 수동 mapping 하지 않고, `unresolvedRecords`로만 추적한다.
- 추후 실제 운영상 중요도가 생기면 그때 원본 주소 교정 또는 별도 검증 루프로 처리한다.
