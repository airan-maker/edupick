# Run Guide And Worklog

> Date: 2026-03-12

이 문서는 오늘 기준 `Edupick` 로컬 실행 방법과 작업 내역을 함께 정리한다.

## 1. 실행 전 준비

- Node.js: `>= 20`
- pnpm: `>= 9`
- Docker Desktop 실행 필요

루트 기준 설치:

```bash
pnpm install
```

## 2. DB / Redis 실행

루트에서 실행:

```bash
pnpm docker:up
```

또는:

```bash
docker compose up -d
```

기본 포트:

- Postgres: `localhost:5433`
- Redis: `localhost:6379`

기본 Postgres 접속 정보:

- DB: `edupick`
- User: `postgres`
- Password: `edupick123`

상태 확인:

```bash
docker compose ps
docker compose logs postgres
```

## 3. DB 마이그레이션 / Prisma

처음 실행하거나 스키마가 바뀌었으면:

```bash
pnpm --filter @edupick/api exec prisma migrate deploy
pnpm --filter @edupick/api exec prisma generate
```

완전 초기화가 필요하면:

```bash
docker compose down -v
docker compose up -d postgres redis
pnpm --filter @edupick/api exec prisma migrate deploy
pnpm --filter @edupick/api exec prisma generate
```

현재 `Role` enum 기준값:

- `PARENT`
- `INSTRUCTOR`

확인 쿼리:

```sql
SELECT unnest(enum_range(NULL::"Role"));
```

## 4. 서버 실행

API 서버 실행:

```bash
pnpm dev:api
```

또는:

```bash
pnpm --filter @edupick/api dev
```

기본 주소:

- API: `http://localhost:4000/v1`
- Swagger: `http://localhost:4000/api/docs`

## 5. 프론트 실행

웹 실행:

```bash
pnpm dev:web
```

또는:

```bash
pnpm --filter @edupick/web dev
```

기본 주소:

- Web: `http://localhost:3000`

## 6. 권장 실행 순서

1. `pnpm install`
2. `pnpm docker:up`
3. `pnpm --filter @edupick/api exec prisma migrate deploy`
4. `pnpm --filter @edupick/api exec prisma generate`
5. 새 터미널 1: `pnpm dev:api`
6. 새 터미널 2: `pnpm dev:web`

## 7. 오늘 Worklog

### 제품 / 역할 구조

- 실제 코드와 DB 기준 역할을 `PARENT`, `INSTRUCTOR` 2개로 정리했다.
- `ADULT`, `ACADEMY_OWNER`는 실제 코드/DB에서 제거하고, 장기 전략 문서에만 남기기로 했다.
- 개발 중에는 `학부모 / 강사` 전환 UI를 유지하되, 배포 전 노출 정책은 다시 결정하기로 했다.

### DB / Prisma

- Docker 기반 Postgres/Redis 실행 구성을 사용하도록 정리했다.
- Postgres `Role` enum이 `PARENT`, `INSTRUCTOR`만 가지도록 reset + migration 기준을 맞췄다.
- Prisma migration 적용 및 client generate 기준을 재정리했다.

### 인증 / 로그인

- 카카오 로그인만 있던 구조에 일반 이메일 회원가입 / 로그인 API를 추가했다.
- `User.passwordHash` 필드를 추가했다.
- 강사 전용 진입 경로에서 카카오 로그인/회원가입 시 신규 계정이 `INSTRUCTOR`로 생성되도록 role intent 전달 구조를 추가했다.
- 로그인 후 실제 `role=INSTRUCTOR`이면 기존처럼 운영 하단 네비가 유지되도록 구조를 유지했다.

관련 파일:

- `apps/api/src/auth/*`
- `apps/api/prisma/schema.prisma`
- `apps/api/prisma/migrations/20260312170000_password_auth/migration.sql`
- `apps/web/src/app/(auth)/login/page.tsx`
- `apps/web/src/app/(auth)/signup/page.tsx`
- `apps/web/src/app/(auth)/kakao/callback/page.tsx`

### 공개 진입 구조

- 학부모 메인은 기본 서비스 진입으로 두고, 강사 화면은 별도 `강사 전용` 경로로 진입하게 정리했다.
- 데스크톱 상단 메뉴와 모바일 햄버거 메뉴에 `강사 전용` 진입을 추가했다.
- 강사용 진입 페이지를 별도 추가했다.

관련 파일:

- `apps/web/src/components/navigation/PublicHeader.tsx`
- `apps/web/src/app/(auth)/instructor/page.tsx`

### 공개 one-page 소개 페이지

- 공개 메인 `/`를 앱 미리보기형 화면에서 분리해 one-page 서비스 소개 페이지로 재구성했다.
- 로그인 전에는 서비스 소개, 역할별 진입, 핵심 흐름만 보여 주고 실제 사용 화면은 로그인 후 홈으로 이어지도록 정리했다.
- 공개 헤더는 페이지별로 섹션형 네비를 받을 수 있게 확장했고, 기본 공개 메뉴에서도 `일정표`처럼 앱 성격이 강한 항목은 제외했다.
- 공개 페이지에 실제 추천 학원, 실제 후기, 운영 기준 FAQ 섹션을 연결했다.
- 추천 학원/후기는 공개 API `GET /academies/highlights`에서 한 번에 내려 주도록 추가했다.

관련 파일:

- `apps/web/src/app/page.tsx`
- `apps/web/src/components/public/PublicLandingDataSection.tsx`
- `apps/web/src/components/navigation/PublicHeader.tsx`
- `apps/web/src/lib/public-faq.ts`
- `apps/api/src/academy/academy.controller.ts`
- `apps/api/src/academy/academy.service.ts`

### 학부모 홈 개선

- 기존 소개형 홈은 앱 홈으로 쓰기 부적절하다고 판단했다.
- 학부모 홈을 `Today Dashboard`형으로 재구성했다.
- 부모 기본 진입을 `/home`으로 이동했다.
- 하단 네비 첫 탭도 `/home`을 가리키도록 변경했다.
- 홈에서 다음 수업, 오늘 수업 수, 읽지 않은 알림, 입금 대기, 빠른 행동, 이번 주 일정, 최근 알림을 바로 보게 했다.
- 홈에 집 근처 추천 학원 데이터를 연결했다.
- 학원 상세를 보면 최근 본 학원이 로컬 히스토리로 저장되고 홈에서 다시 열 수 있게 했다.
- 자녀 없음 / 수강 없음 상태에 맞는 빈 상태 CTA도 추가했다.

관련 파일:

- `apps/web/src/app/(app)/home/page.tsx`
- `apps/web/src/app/(app)/discover/[academyId]/page.tsx`
- `apps/web/src/components/ui/BottomNav.tsx`
- `apps/web/src/lib/recent-academies.ts`
- `apps/web/src/lib/role-ui.ts`
