# 역할 전환 Todo

## 목적

- 개발 중에는 하나의 카카오 계정으로 `학부모`, `강사`, `학원장` 화면을 모두 검증할 수 있어야 한다.
- 운영 전에는 일반 사용자에게 역할 전환 UI가 노출되지 않도록 정리해야 한다.

## 개발 중 유지 항목

- [ ] `마이페이지 > 설정`의 역할 전환 UI를 유지한다.
- [ ] 개발 계정 1개로 `PARENT`, `INSTRUCTOR`, `ACADEMY_OWNER` 화면 접근을 모두 검증한다.
- [ ] 온보딩에서 `ACADEMY_OWNER` 선택지가 정상 저장되는지 확인한다.
- [ ] 어드민 페이지 구현 후 역할별 접근 경로와 화면 렌더링을 다시 점검한다.

## 배포 전 정리 항목

- [ ] 역할 전환 UI를 제거하거나 `dev/staging` 환경에서만 보이도록 제한한다.
- [ ] 운영 환경에서는 일반 사용자 UI가 아니라 관리자/백오피스 경로로만 역할 변경이 가능하게 한다.
- [ ] 배포 체크리스트에 `역할 전환 UI 비노출` 항목을 추가한다.

## 향후 설계 결정

- [ ] 운영 정책을 결정한다: `단일 역할 유지` 또는 `다중 역할 + activeRole`.
- [ ] 다중 역할이 필요하면 `user.role` 단일값 대신 `roles` 또는 `memberships` 구조로 마이그레이션 설계를 진행한다.
- [ ] 역할 전환 후 권한 반영 방식을 정리한다: 재로그인, 토큰 재발급, 또는 active role 전환 방식.

## 현재 판단

- 현재 역할 전환 UI는 개발 편의를 위한 임시 장치로 유지한다.
- 운영 배포 전에는 숨기거나 제거하는 방향으로 간다.
